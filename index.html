装饰者模式 Decorator Pattern

可以在运行时动态添加附加功能到对象中，而不改变原本对象的设计模式.

策略模式 Strategy Pattern

代码的客户端可以用同一个接口，同时根据上下文从多个算法中选择一个用于处理特定任务的算法。

代理模式 Proxy Pattern

一个对象充当另一个对象的接口

中介者模式 Mediator Pattern

让一个对象充当各个对象相互通信的中介者，以促进各个对象的松耦合，提高可维护性。

观察者模式（发布订阅模式） Observer Pattern

当一个对象发生了事情后，发布者会通知所有订阅者。
