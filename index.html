装饰者模式 Decorator Pattern

装饰者模式是一种动态地往一个类中添加新的行为的设计模式。就功能而言，修饰模式相比生成子类更为灵活，这样可以给某个对象而不是整个类添加一些功能。

  • + 比静态继承更灵活
  • + 避免类变得太过复杂
  • - 有许多小对象，难以学习

策略模式 Strategy Pattern

  • + 抽象算法，便于重用
  • + 消除了条件语句

策略模式指的是代码的客户端可以用同一个接口，同时根据上下文从多个算法中选择一个用于处理特定任务的算法。

代理模式（虚拟代理） Proxy Pattern

代理模式是指一个类可以作为其它东西的接口。代理者可以作任何东西的接口：网络连接、存储器中的大对象、文件或其它昂贵或无法复制的资源。

  • + 虚拟代理可以进行最优化

中介者模式 Mediator Pattern

中介者模式是让一个对象充当各个对象相互通信的中介者，以促进各个对象的松耦合，提高可维护性。

  • +减少了子类生成
  • +将各个对象解耦
  • +简化了对象协议
  • +对对象如何协作进行了抽象
  • -中介者自身可能会变得难以维护

观察者模式（发布订阅模式） Observer Pattern

观察者模式是指由一个目标对象管理所有依赖于它的观察者对象，并且在它本身的状态改变时主动发出通知。

  • +目标和观察者之间的抽象耦合
  • +可以随时添加删除观察者
